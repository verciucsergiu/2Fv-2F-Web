4.Technologies Used

Node.js

Both our Front-End and Back-End are built on Node.js

Code Bird

We use codebird to communicate with Twitter

Super Agent

We use superagent to communicate with GitHub,LinkedIn,Facebook and GitBucket

Webpack

We use Webpack to package our front-end app

bcrypt

We hash all passwords using bcrypt. Passwords will never be visible to the admin or to the user

jwt

We secure our sessions using jwt.We store them in the local storage and remove them upon Logout

typeorm

We use typeorm for the database-server connection. Trough typeorm we are safe against any type of SQL Injection, but we still check for any type of scripts/injections in our request handlers.

Dependency Injection

Trough our framework we've built Injectable classes which make the developing a lot easier. Some notable are:
    Controller functionality
  • HttpPost
  • HttpPut
  • HttpGet
  • HttpDelete
    • Repositories
  • StudentRepository
  • TeacherRepository
  • AttendanceRepository
  • PendingInvitesRepository
  • GroupRepository
    • Commands/Querries
  • CommandHandler
  • QueryHandler
    • Utils
  • JwtFactory
  • UserService

• CQRS

We use CQRS to completely separate responsability between actions that change data and actions that only retrieve them, all the way from the rest controller

5.Back-End Development

As the front-end, the back-end is also build on node.js, however we also use TypeScript since it compiles to JavaScript but it has more functionalities.

Design-patterns:

• Factory

Mainly we use the Factory Pattern to build jwt's

• Bridge

ALmost all our important classes follow specific interfaces

• Repository

For every entity we have a different repository extended from the Base Repository, which we inject when needed

• Builder

We use WebApiBuilder to build the main application and start it.
